Haldex Calibration

Generation

Must match the Haldex controller fitted to the car - it selects the CAN decode and command format. MQB cars (e.g. Mk3 TT, 8V S3, MQB Golf R) are Gen 5 (0CQ); the 0AY variant is the PQ-derived unit. Wrong generation means no lock control.

Learn Haldex

Sweeps correction factors 0–100% and records actual Haldex engagement at each step, building a calibration table that replaces the formula. Car must be running with Haldex CAN active.

Turn this on if the rear lock surges or won't hold steady at the part-lock modes (60:40, 75:25). Only the 554K Haldex controller needs it - leave it off otherwise.

Full-lock calibration

220 Nm

A per-car tuning number for Fix Hunting mode - NOT your engine's torque. There's a correct value where the lock you ask for matches what the Haldex actually delivers (1:1). Higher does not lock harder: set it too high and the coupling grabs early and slams shut. Find your value with a road-test sweep, don't guess. 220 is a safe starting point. Only does anything while Fix Hunting is on (or during a Learn).

Ext. Control

Press: each tap advances to the next mode. Hold: button held forces the selected mode; releasing reverts.

Force Mode Priority

When multiple triggers are active simultaneously, the highest-priority one wins.

Brake / Handbrake Outputs

Drive the controller's brake/handbrake output pins from the matching input. "Follow" mirrors the input; "Invert" flips it (output active when the input is released). Leave off unless your wiring uses these pins.

Controller Options

Controller & Connectivity

Hard passthrough. The unit forwards every chassis frame to the Haldex untouched - no mode, no lock map, no overrides. The kill-switch when you want the car fully stock without unplugging.

For a car with no chassis CAN feeding the controller. It ignores incoming chassis frames and generates its own, so it can drive the Haldex on a bench or a stripped car. Stock mode is unavailable here (nothing to mirror).

Standalone only. Still read real chassis CAN (speed, throttle) when it is present, so Expert mode's speed/throttle map works instead of running blind.

Emit OpenHaldex status on the bus and accept mode-change commands from another OpenHaldex or external controller over CAN. Leave off if nothing else on the bus talks to it.

Diagnostics (UDS)

Gen 5 (MQB) only. Polls the Haldex ECU directly for live diagnostic data. Requires Haldex CAN healthy.

Note: Will Disable Diag. Comms, Disable if Diag Req.

Buttons

Low-Power Sleep

After 5 min idle with no WiFi clients and both CAN buses silent, sleep until a CAN frame arrives. The first wake frame may be lost.

Additionally parks the CAN transceivers in standby while sleeping, drops CPU floor to 10MHz, and lowers WiFi AP TX power. Bus activity wakes the transceivers via GPIO interrupt within microseconds; a 2s safety-net probe runs as backup.

Lock Response Times (ms)

How long the lock target takes to ramp up (attack) and decay (release) on changes - time for a full 0-100% travel. Higher = slower. 0 = instant in that direction. Disable rate limiting for instant transitions both ways (legacy behaviour).

Steering Gain Taper

Reduces lock as steering angle grows so the rear axle is not fighting the front through tight corners (driveline windup). Gain stays 100% below the start angle and ramps down to the minimum at the full angle. Needs MQB steering-angle data; a stale or degraded signal leaves the gain at 100%.

Start angle

45 deg

Full angle

180 deg

Minimum gain
